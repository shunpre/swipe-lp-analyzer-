# Swipe LP Auto-Analyzer Prototype

スワイプLPのイベントデータを分析し、AIが次手を提案する自動分析ツールのプロトタイプです。

## 🎯 目的

このプロトタイプは、以下の自動フローの実現可能性を検証し、ユーザーとの認識齟齬を確認するために作成されています。

1. **データ取得**: BigQueryからイベントデータを取得
2. **可視化**: グラフでデータを可視化
3. **画像表示**: スワイプLPの各ページ画像を表示
4. **AI分析**: 現状分析と次手提案をAIが自動生成
5. **インタラクション**: アクションボタンとフリーチャット機能

## 🚀 起動方法

### 前提条件
- Python 3.8以上
- pip

### インストール

```bash
pip install -r requirements.txt
```

### 実行

```bash
streamlit run app/main.py
```

または

```bash
./run.sh
```

アプリケーションは `http://localhost:8501` で起動します。

## 🎨 機能概要

### タブ1: 📈 分析
- **ページ別パフォーマンス**: ビュー数とコンバージョン率を表示
- **時系列推移**: ビュー数と離脱率の推移をグラフ化

### タブ2: 🔍 詳細
- **ページ画像**: スワイプLPの各ページ画像を表示（プレースホルダー）
- **詳細メトリクス**: 選択したページの詳細データを表示

### タブ3: 💡 提案
- **現状分析**: ボトルネックを自動検出
- **推奨アクション**: A/Bテスト、コンテンツ改善、詳細分析のボタン

### タブ4: 💬 チャット
- **フリーチャット**: AIに自由に質問できるインターフェース

## 📊 ダミーデータについて

現在、プロトタイプはダミーデータを使用しています。

**実装予定:**
- BigQuery APIの仕様確定後、実際のデータ取得に置き換え
- スワイプLPのURL解析機能を追加し、画像URLを動的に取得

## 🤖 AI機能について

現在、AI分析・提案ロジックはダミー実装です。

**実装予定:**
- OpenAI API（GPT-4など）を統合
- 分析結果に基づき、自然言語で考察・仮説・提案を生成
- ユーザーの質問に対し、データに基づいた回答を生成

## 🔄 次のステップ

1. **データ取得APIの仕様確認**: BigQueryからのデータ取得方法を確定
2. **LP画像URL抽出**: スワイプLPのURL構造を解析し、画像URLを取得
3. **AI分析ロジックの実装**: OpenAI APIを統合し、分析・提案を自動生成
4. **UI/UXの調整**: ユーザーフィードバックに基づき、インターフェースを改善
